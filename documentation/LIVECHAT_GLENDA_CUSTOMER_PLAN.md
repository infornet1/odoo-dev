# Glenda Live Chat — Customer Frontend Plan

**Status:** Planned | **Priority:** Next after OdooBot bridge | **Estimated effort:** 1 day  
**Depends on:** `ueipab_ai_agent` v17.0.1.40.3 (OdooBot bridge already deployed)

---

## What This Is

Extend the existing Glenda OdooBot bridge so that **parents, students and prospects** can
chat with Glenda directly from the school website (`ueipab.edu.ve`) through a floating
chat bubble — without needing WhatsApp or an Odoo account.

```
Parent on website
      ↓
Clicks chat bubble (im_livechat widget)
      ↓
discuss.channel created (channel_type = 'livechat')
      ↓
_get_answer() override fires (our existing bridge, extended)
      ↓
Claude Haiku → Glenda responds with institutional knowledge
      ↓
Customer sees reply in the chat bubble
```

---

## Why `im_livechat` (Not a Custom Widget)

| Option | Effort | Why |
|--------|--------|-----|
| `im_livechat` (Odoo module) | Low | Already in Odoo Community, free, zero new infrastructure. Our bridge is 95% done — just extend `channel_type` check |
| Custom JS widget + controller | High | Full build: HTTP endpoint, JS chat UI, session management, history — not worth it |
| WhatsApp only | Done | Already live, but requires the customer to have WhatsApp and initiate manually |

## Current Bridge vs Extended Bridge

```python
# CURRENT (staff only — private DM with OdooBot)
if record.channel_type == 'chat':
    → call Claude

# EXTENDED (staff + customers)
if record.channel_type in ('chat', 'livechat'):
    → call Claude
```

One line change in `mail_bot_glenda.py`. Everything else (knowledge, history,
guards, fallback) is identical.

## Implementation Steps

### Phase 1 — Install `im_livechat` (30 min)

```bash
# On production Odoo container
docker exec ueipab17 /usr/bin/odoo -d DB_UEIPAB -i im_livechat \
  --stop-after-init --http-port=18069
docker restart ueipab17
```

Configure in Odoo UI:
- **Live Chat → Channels → New**: name "Colegio Andrés Bello", add OdooBot as operator
- **Chatbot**: leave empty — our `_get_answer` override handles responses
- **Channel widget**: copy the generated `<script>` tag

### Phase 2 — Extend the bridge (20 min)

In `mail_bot_glenda.py`, change one condition:

```python
# Before
if not (hasattr(record, 'channel_type') and record.channel_type == 'chat'):

# After
if not (hasattr(record, 'channel_type') and record.channel_type in ('chat', 'livechat')):
```

Adjust system prompt for livechat context (visitor is external, not staff):

```python
def _glenda_system_prompt(self):
    channel_type = self.env.context.get('channel_type', 'chat')
    if channel_type == 'livechat':
        persona = (
            "Eres Glenda, asistente virtual del Instituto Privado Andrés Bello. "
            "Estás atendiendo a un visitante del sitio web del colegio — puede ser "
            "un representante, prospecto o estudiante. No conoces su identidad. "

        persona = (
            "Eres Glenda, asistente virtual del Instituto Privado Andrés Bello, "
            f"respondiendo preguntas del personal interno. Hablas con {self.env.user.name}."
        )
    ...
```

### Phase 3 — Embed on school website (15 min)

Paste the `<script>` tag generated by Odoo's livechat channel into the school website's
`<head>` or before `</body>`. The floating bubble appears automatically on every page.

```html
<!-- Example — actual tag generated by Odoo -->
<script src="https://odoo.ueipab.edu.ve/im_livechat/loader/1" ...></script>
```

## What Customers Experience

1. Visit `ueipab.edu.ve` → see a **chat bubble** (bottom right)
2. Click → chat opens: "Hola, soy Glenda, asistente del Colegio Andrés Bello. ¿En qué puedo ayudarte?"
3. Ask about fees, enrollment, policies → get real-time answers
4. For billing/debt questions → Glenda asks for cédula → hands off to pagos@

---

## Behaviour Differences: Staff vs Customer

| Aspect | Staff (OdooBot Discuss) | Customer (Livechat) |
|--------|------------------------|---------------------|
| Identity | Logged-in Odoo user (name known) | Anonymous visitor |
| Greeting | "Hola [name], ¿en qué te ayudo?" | "Hola, soy Glenda…" + ask name |
| Balance lookup | No (internal context) | No (not identified) |
| ACTION: markers | Stripped | Stripped |
| Flyers | No (text only) | No (text only in widget) |
| History | Last 10 channel messages | Last 10 channel messages |
| Fallback | Default OdooBot | Default OdooBot welcome |

---

## Guards & Limits

| Guard | Behaviour |
|-------|-----------|
| `ai_agent.dry_run = True` | Skips Claude, falls back to default OdooBot |
| `ai_agent.credits_ok = False` | Blocked by `claude_service` |
| Claude API error | Falls back to default OdooBot silently |
| No response content | Falls back to default OdooBot |
